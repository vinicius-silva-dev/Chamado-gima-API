## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

## Deployment

When you're ready to deploy your NestJS application to production, there are some key steps you can take to ensure it runs as efficiently as possible. Check out the [deployment documentation](https://docs.nestjs.com/deployment) for more information.

If you are looking for a cloud-based platform to deploy your NestJS application, check out [Mau](https://mau.nestjs.com), our official platform for deploying NestJS applications on AWS. Mau makes deployment straightforward and fast, requiring just a few simple steps:

```bash
$ npm install -g @nestjs/mau
$ mau deploy
```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.

## Casos de uso
- [x] Deve ser possível criar um usuário;
- [] O usuário criado deve conseguir logar no sistema;
- [x] O usuário deve conseguir criar um chamado;
- [] Na criação do chamado, o usuário deve conseguir anexar arquivos;
- [] Com o chamado criado, deve ser possível adicionar novas atualizações;


## Regras de negócio
- [x] Deve ser possível criar um usuário, por padrão, o usuário será padrão.
- [x] O analistas serão criados via banco.
- [x] No sistema de chamados, tanto o usuário padrão como os analistas, deve conseguir logar.
- [x] Se o usuário padrão esquecer a senha, o mesmo deve conseguir recuperar a senha.

- [x] O usuário padrão deve conseguir criar um novo chamado.
- [x] Na criação do chamado, o usuário deve escrever no mínimo 25 caracteres na descrição do chamado.
- [x] Cada usuário padrão terá acesso a ver e editar apenas os chamados que o mesmo abriu.
- [x] O usuário padrão pode cancelar o chamado mediante a uma justificativa.
- [] Se o chamado for cancelado, não pode ser revertido via sistema, o usuário terá que abri um novo. (FRONT-END)
- [] Todos os campos, na abertura do chamado, são obrigatórios preencher.(FRONT-END)
- [] Por enquanto, os anexos serão apenas fotos.
- [x] As atualizações do chamado, pode conter anexos.

- [x] O analísta tem acesso a ver, editar e a opção de atender todos os chamados em aberto.
- [x] O analista pode filtrar os chamados que estão ou foram atendidos por ele.
- [x] O analista não pode cancelar o chamado sem apresentar uma justificativa.
- [x] O analista pode criar um novo chamado.
- [] O analísta/adm tem acesso a ver todos os usuários cadastrados, editar e deletar os usuários. (FRONT-END)
